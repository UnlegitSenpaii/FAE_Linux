# Contributing to FAE_Linux

Thanks for your interest in contributing! Please read the guidelines below before opening a pull request.

## What is welcome

- **Pattern updates** — if Factorio updates break the patcher, updated byte patterns are the most valuable contribution.
- **Bug fixes** — small, focused fixes with a clear description of what they address.
- **Minor functionality improvements** — improvements that are easy to understand and review.

## What will be rejected

- **Major cosmetic changes** — reformatting, restructuring, or style-only changes to the codebase.
- **Large, sweeping PRs** — pull requests that touch many files or make broad changes will be closed without comment due to security and maintainability concerns. Keep changes small and focused.

## How to contribute

1. **Fork** the repository and create a branch from `master`.
2. **Make your changes** — keep the diff small and on-topic.
3. **Test** that the patcher still works against a current Factorio binary.
4. **Open a pull request** with a clear title and description of what changed and why.

## Security

If you discover a security vulnerability, see [SECURITY.md](SECURITY.md) for how to report it. Sensitive issues should be disclosed privately via Discord rather than as a public issue.
